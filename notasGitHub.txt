Algunos comandos que pueden ayudar cuando colaboren con proyectos muy grandes de github:

git log --oneline - Te muestra el id commit y el título del commit.
git log --decorate- Te muestra donde se encuentra el head point en el log.
git log --stat - Explica el número de líneas que se cambiaron brevemente.
git log -p- Explica el número de líneas que se cambiaron y te muestra que se cambió en el contenido.
git shortlog - Indica que commits ha realizado un usuario, mostrando el usuario y el titulo de sus commits.
git log --graph --oneline --decorate y
git log --pretty=format:"%cn hizo un commit %h el dia %cd" - Muestra mensajes personalizados de los commits.
git log -3 - Limitamos el número de commits.
git log --after=“2018-1-2” ,
git log --after=“today” y
git log --after=“2018-1-2” --before=“today” - Commits para localizar por fechas.
git log --author=“Name Author” - Commits realizados por autor que cumplan exactamente con el nombre.
git log --grep=“INVIE” - Busca los commits que cumplan tal cual está escrito entre las comillas.
git log --grep=“INVIE” –i- Busca los commits que cumplan sin importar mayúsculas o minúsculas.
git log – index.html- Busca los commits en un archivo en específico.
git log -S “Por contenido”- Buscar los commits con el contenido dentro del archivo.
git log > log.txt - guardar los logs en un archivo txt

Ahora que esta todo bien configurado hay que supervisar las etiquetas y versiones. Primero que nada el comando para ver el historico de cambios de una manera compacta es:
git log --all --graph --decorate --oneline

Esta cadena de comandos puede ser resumida creando un "alias" dentro de Linux. Lo cuál permite sintetizar una linea de comando en un solo termino. Ejemplo:
alias historico="git log --all --graph --decorate --oneline"

El alias introducido quedará guardado en un entorno local, solo hay que tener precaución de recordar éste detalle y que hay que volver a crear dichos alias al cambiar a otro entorno de trabajo.


Respecto a los "tags". Son etiquetas de "versión" que uno añade a los commits para tener contról y organización en el proyecto.
La linea de comando utilizada para ésto sería la siguiente:
git tag -a v0.1 -m "Mensaje de la versión" (Clave del commit al que querramos añadir el tag)

Una vez hecho ésto podrá consultarse que tags hay disponibles en el proyecto usando el comando:
git tag
